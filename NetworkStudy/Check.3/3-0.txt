■ ネットワーク層
IP...インターネットプロトコル。通信するためのルール
IPヘッダ...IPパケットに付与される情報
ICMP...internet Contorol Message Protocol
      TCP/IPでネットワークの疎通がされているノード間で、通信状態の確認をする為に使われるプロトコル
ICMPメッセージ...問い合わせやエラーに関してメッセージでやり取りすることで
　　　　　　　　インターネット通信においてさまざまな役割を果たす
IPアドレス...ネットワーク上の機器に割り当てられるインターネット上の住所
オクテット..８ビットずつカンマで区切られた部分の事。左から順に第１、第２、第３、第４オクテットと呼ぶ。




ネットワーク部...ネットワークを識別するアドレス
ホスト部...コンピュータや各機器を識別するアドレス。個々のホストに割り当てられるアドレス
クラス...ネットワーク部とホスト部の境界を、IPv4アドレスの上位数ビットによって決定するもの
プライベートIPアドレス...社内LANなど小規模のネットワークで割り当てるIPアドレス
グローバルIPアドレス...インターネットに接続されているコンピュータや通信機器を
　　　　　　　　　　　　個々に特定するための割り当てられたIPアドレス
ネットワークアドレス...ホスト部の２進数のビットがすべて０のアドレス
ブロードキャストアドレス...ホスト部の２進数のビットがすべて１のアドレス
ループバックアドレス...ネットワークアドレスとして使用禁止されている自分自身を示す仮想的なIPアドレス
サブネット...大きいネットワークを適切なサイズに分割して運用IPアドレスのネットワーク部のビット数を増やし
 　　　　　　ホスト部のビット数を減らす

サブネットマスク...３２ビットで構成。IPアドレスとセットで使う。ネットワークの範囲を指定する

プレフィックス表記...サブネットマスクを略したもの　「/」
デフォルトゲートウェイ...同一ネットワーク内のノードから外部ネットワークに属するノードと
　　　　　　　　　　　　通信する際に出入り口になるポート。各ノードに設定する。
　　　　　　　　　　　　同一ネットワークに該当するルータのポートが当てはまる
　　　　　　　　　　　　
IPv6...各コンピュータに割り振られるIPアドレスを１２８ビットのデータとしてあらわす。
　　　 最大約３４０澗台のPCが接続できる
サブネットプレフィックス...IPv6のネットワーク部、前半６４ビットがサブネットプレフィックス
インターフェイスID　IPv6のホスト部後半、後半６４ビット。MACアドレスから算出
ルータ...コンピュータネットワークの中で２つ以上の異なるネットワークにデータを中継する通信機器
         ルート選択機能をもつ。外部ネットワークとの出入り口になる。
ルーティングテーブル...コンフィギュレーションモードで見れる。ルーティングの情報が記載されたもの
ネクストホップ...次に転送する最も近いルータやインターフェースのこと
ARPテーブル...ARP（Address Resolution Protocol）によって割り出したIPアドレスと
              MACアドレス（EthernetやWi-Fiなどの物理アドレス）の対応表。
ARP...IPアドレスからEthernetのMACアドレスの情報を得られるプロトコル
コアルータ...基幹ネットワークにおいてメインで用いられるルータ
　　　　　　通信キャリアやISPネットワークの通信を中継するために利用されるルータで
　　　　　　大量のデータを転送する必要があるため、高速なデータ転送が可能
センタールータ...企業で中心的に利用されるルータ
　　　　　　　　拠点となる本社や、データセンター等の企業内で中心となる場所を
　　　　　　　　接続するためのネットワークを構成する
エッジルータ...企業で利用されるルータ
　　　　　　　　中心となる拠点を除く、営業所や支社等の拠点間を接続するネットワークを構成する
ブロードバンドルータ...家庭環境やSOHO環境等、インターネット回線に接続することを目的として利用されるルータ
ルーティング...異なるネットワークにパケットを送信する際に最適な経路を決めること
直接接続...ルータが直接接続しているネットワークのルート
スタティックルーティング...管理者が宛先ネットワークへの最適なルートを手動で設定したルート
ダイナミックルーティング...ルーティングプロトコルにより動的に追加されるルート

～ダイナミックルーティングの種類～
ディスタンスベクタ型...ホップ数がメトリック。小型ネットワーク向き
リンクステート型...各ルータが全体の接続情報をみて最適ルートを選択。大型ネットワーク向き
ハイブリッド型...ディスタンススベクタ型をベースに、リンクステート型を組み合わせた方式
　　　　　　　　　ひとつでも非シスコ社デバイスがあるとできない



■ トランスポート層
コネクション型通信...データ通信において、通信を開始する前に相手との間で
　　　　　　　　　　　仮想的な専用通信路（コネクション）を確立し、
　　　　　　　　　　　それを通じてデータの送受信を行う通信方式
ACK...二者間の通信で、相手方に何らかの肯定的な応答を返す際に送られる信号やデータ、パケットなどのこと
コネクションレス型通信...Datagram（データグラム）とも呼ばれる、経路確保などの事前のやり取り（コネクションの確立）
                         無しに、データ送信を開始するパケット通信の方式である。 TCP/IPにおけるUDPで用いられている。
TCP...1対1のセッションによる信頼性の高い通信を行うためのプロトコル
スリーウェイハンドシェイク...TCPにおいて使用されるコネクションの確立のことを3ウェイハンドシェイクという
Telnet...IPネットワークにおいて、遠隔地にあるサーバやルーター等を端末から操作する通信プロトコル
HTTP1...Hypertext Transfer Protocol
        WebサーバとWebブラウザの間で、Web情報をやりとりするためのプロトコル
FTP...File Transfer Protocol
      
コンピュータネットワーク上のクライアントとサーバの間で
      ファイル転送を行うための通信プロトコル
SMTP...Simple Mail Transfer Protocol
      インターネットなどのTCP / IPネットワークで標準的に用いられる、メールを送信するための通信プロトコル
TCPヘッダ...TCPセグメントに付与される情報
UDP...User Datagram Protocol
　　　インターネット・プロトコル(IP)を使ったネットワークにおいて
　　　アプリケーション同士が最小限の仕組みでデータを送受信できるように考案された、 シンプルなプロトコル
RTP...Real time Transport Protocol
      音声や映像のデジタルデータによって構成されたIPパケットで
      受信側で音声や映像の再生を実現させるためにUDPの上位層で動作するプロトコル
RTCP...RTPなどを用いてデータをリアルタイム伝送する際に、送受信制御などを行うプロトコル
DHCP...Dynamic Host Configuration Protocol
       クライアントに対して動的にIP設定パラメーターを提供する機能
       DHCPサーバーは、クライアントの要求に対して、あらかじめプールされたIPアドレスの中から
       使用されていないアドレスを選び、一定期間クライアントに割り当てます。
       
TFTP...UDPを使った軽量なファイル転送プロトコル
       Windowsでは、RISやPXEサービスといった、ネットワーク・ブート環境で利用される
       Windows Server OSにはtftpのサーバ・プログラムtftpd.exeが用意されているので
      これを使ってtftpサーバを構築できる
RIP...Routing Information Protocol
      ネットワークで接続したルータ間で使用するルーティングプロトコル
      各ルータはRIPを使用して自ルータから到達できるネットワークと
     そのネットワークへのホップ数(メトリック)を通知し合うことによって経路情報を生成する
     
UDPヘッダ...UDPセグメントに付与される情報。
